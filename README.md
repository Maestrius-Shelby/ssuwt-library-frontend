## 🚀 Деплой и запуск веб-приложения SSUWT Library

### 📦 Требования к серверу

Linux-сервер должен быть подготовлен следующим образом:

* ✅ Установлена ОС Ubuntu 20.04 или выше
* ✅ Пользователь с правами `root` или `sudo`
* ✅ Открытые порты `80` и `443`, если планируется HTTPS
* ✅ Установлены:

  * `git`
  * `docker`
  * `docker-compose`

---

### 🔧 Настройка окружения (`.env`)

Для работы приложения необходимо создать файлы окружения на основе примеров.

# Копируем примеры конфигов
```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
```

После этого отредактируйте `backend/.env` и `frontend/.env` — укажите реальные значения переменных (пароли, ключи, URL).

> ⚠️ **Важно**  
> Файлы `.env` добавлены в `.gitignore` и **не попадают в репозиторий**.  
> Это сделано для безопасности: секретные данные не должны быть доступны посторонним.

### 🔑 Подключение к серверу

Подключаемся к серверу по SSH:

```bash
ssh root@<IP-адрес сервера>
```

Далее вводим пароль администратора.

---

### 🛠 Установка необходимых компонентов

Обновляем список доступных пакетов:

```bash
sudo apt update
```

Устанавливаем Git, если он ещё не установлен:

```bash
sudo apt install git-all -y
```

Клонируем репозиторий проекта через SSH:

> ⚠️ **Убедитесь, что SSH-ключ GitHub добавлен в сервер**

```bash
git clone git@github.com:dannn8/SSUWT-library.git
```

Переходим в директорию проекта:

```bash
cd SSUWT-library
```

---

### 🐳 Установка Docker и Docker Compose

Устанавливаем утилиты для добавления внешнего репозитория Docker:

```bash
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
```

Добавляем официальный GPG-ключ Docker:

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```

Добавляем Docker-репозиторий в список источников пакетов:

```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

Обновляем список пакетов и устанавливаем Docker:

```bash
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y
```

Проверяем, что Docker установлен:

```bash
docker --version
```

---

### 🚀 Запуск приложения

Находимся в корне проекта. Собираем и запускаем контейнеры с использованием production-профиля:

```bash
docker compose --profile prod up --build
```

#### Параметры запуска

| Параметр         | Описание                                                 |
| ---------------- | -------------------------------------------------------- |
| `--build`        | Пересобирает образы при необходимости                    |
| `--profile prod` | Запускает только сервисы, указанные в production-профиле |

---

### 🛑 Остановка приложения

Для остановки и удаления всех контейнеров:

```bash
docker compose --profile prod down
```

### 🔄 Обновление приложения после изменений

Если в проекте были внесены изменения (например, в исходном коде фронтенда или конфигурациях), выполните следующую последовательность команд.

#### 1️⃣ Остановить все контейнеры

```bash
docker compose --profile prod down
```

#### 2️⃣ Удалить старый том с билдом фронтенда

> ⚠️ **Обратите внимание:** это удалит собранную версию React-приложения из volume.

```bash
docker volume rm ssuwt-library_frontend_build
```

#### 3️⃣ Повторно запустить и пересобрать все контейнеры

```bash
docker compose --profile prod up --build
```

> 💡 **После выполнения этих команд контейнеры будут использовать свежесобранную версию фронтенда и все обновления вступят в силу.**
